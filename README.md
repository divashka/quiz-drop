# 📑 Техническое задание: Конструктор тестов "QuestDrop"

## 1. Обзор проекта
**QuestDrop** — это SPA-приложение для создания, редактирования и прохождения интерактивных тестов. Проект демонстрирует навыки работы со сложными динамическими формами, управления вложенным состоянием и построения модульной архитектуры.

---

## 2. Технологический стек (Stack & Tooling)

| Слой | Технология | Обоснование |
| :--- | :--- | :--- |
| **Framework** | React 18+ (Vite) | Быстрая сборка и современный API (хуки, саспенс). |
| **Language** | TypeScript | Строгая типизация моделей данных теста (Question, Answer, Quiz). |
| **State** | Redux Toolkit | Централизованное управление структурой теста и ответами пользователя. |
| **Architecture** | **FSD (Feature-Sliced Design)** | Разделение бизнес-логики на слои для предотвращения "спагетти-кода". |
| **Styling** | Tailwind CSS + Headless UI | Быстрая верстка адаптивных интерфейсов без написания лишнего CSS. |
| **Testing** | React Testing Library | Тестирование ключевых пользовательских сценариев (создание вопроса, подсчет баллов). |

---

## 3. Архитектура (FSD Structure)
Проект должен быть строго разделен на слои:
* `app/`: инициализация Store, глобальные стили, роутинг.
* `pages/`: страницы `EditorPage` (админка), `QuizPage` (прохождение), `ResultsPage`.
* `widgets/`: крупные блоки, например `QuestionStepper` или `EditorSidebar`.
* `features/`: изолированные действия: `add-question`, `delete-answer`, `calculate-score`, `toggle-timer`.
* `entities/`: бизнес-сущности: `quiz` (слайс в Redux), `question` (интерфейсы и UI-карточки).
* `shared/`: переиспользуемые UI-компоненты (Button, Input), API-клиент, константы.

## 4. Функциональные требования

### 4.1 Модуль конструктора (Admin Mode)
1.  **Управление тестом:** создание заголовка, описания и установка лимита времени.
2.  **Динамический список вопросов:**
    * Добавление вопросов трех типов: `Single Choice` (один из многих), `Multiple Choice` (несколько из многих), `Text Input` (ввод фразы).
    * Редактирование вариантов ответов и назначение правильных.
3.  **Валидация (Zod/RTK):** проверка, что в вопросе выбран хотя бы один правильный ответ перед сохранением.
4.  **Drag-and-Drop:** изменение порядка вопросов (библиотека `@dnd-kit/core`).

### 4.2 Модуль прохождения (User Mode)
1.  **Quiz Engine:** пошаговое отображение вопросов (один вопрос на экран).
2.  **Система навигации:** кнопки "Назад/Вперед", сохранение ответов при переходах.
3.  **Таймер:** обратный отсчет с автоматической отправкой теста при завершении времени.

### 4.3 Модуль аналитики
1.  **Алгоритм проверки:** расчет баллов с учетом веса вопросов (если реализовано).
2.  **Детальный отчет:** визуализация ответов пользователя в сравнении с правильными.

## 5. Модель данных (TypeScript Interfaces)

```typescript
export type QuestionType = 'single' | 'multiple' | 'text';

export interface AnswerOption {
  id: string;
  text: string;
  isCorrect: boolean;
}

export interface Question {
  id: string;
  type: QuestionType;
  title: string;
  options?: AnswerOption[];
  points: number;
}

export interface QuizState {
  title: string;
  questions: Question[];
  userAnswers: Record<string, any>; // id вопроса -> ответ
  status: 'idle' | 'editing' | 'playing' | 'finished';
}
```
